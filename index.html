Dissertation Project

As a BSc Computer Science student, I dedicated a significant amount of time and effort towards my dissertation project, which culminated in the creation of a web-based multiplayer game based on the popular card game Schotten Totten.

This project was not just a fun side project, but a challenging academic endeavor that allowed me to apply the knowledge and skills I gained throughout my studies. To implement the game, I utilized the Phaser game engine for the client-side implementation and Node for the server-side implementation. Communication between the server and client was established using Socket.io, a powerful tool for real-time bidirectional event-based communication.

The game follows a client-server architecture, allowing two players to connect with each other and compete against each other in real-time. This required me to overcome several technical challenges, including handling game state synchronization, implementing game rules and logic, and ensuring a smooth and enjoyable user experience.

One of the key achievements of this project was the successful deployment of the game on a live server, which allowed me to test the game under real-world conditions and ensure that it was scalable and could handle multiple concurrent users. This project has given me valuable experience in web development, game design, and networking, and I am proud of the final product that I have created.

Bootcamp 2020: Hangman game

This is my first Processing project summarizing the 2-week Programming Bootcamp at City University (2020).

The program, to run properly uses multiple Processing tools like classes, loops, conditions, methods and randomness. This game has the option to be played by one person or two - this can be chosen after the start of the program by choosing a correct button. Choosing a singleplayer mode will randomly choose the guessing word from the provided text file and start a game. Multiplayer mode will first allow you to input the word that later will be guessed by another player.
When the game is played, each keypress starts the sequence of methods that determine whether the inputted letter is right or wrong, and acts accordingly. If the letter is correct, the program will display it. If not, then the part of the gibbet wil be drawn.
The game ends when the whole word is guessed or the whole gibbet is displayed on the screen. Then the player has a choice to play again with a different word or to exit.

Second year projects
Computer Networks - DISTTER - Distributed Social Network

This project was a coursework concluding my Computer Networks module at City University. The assingment was to create a program that works as a peer-to-peer distributed social network where nodes store and exchange posts.

This program was supposed to be done in Java and send data through a TCP stream. The main purpose of the program was to create a network of nodes that could store and distribute posts. Nodes could either be a TCP client or a TCP server, however after establishing a connection they worked on a peer-to-peer basis. Those nodes communicate to via a number of requests each of which may have a response.

As there is constant communication between two peers in form of sending and receiving requests, to avoid deadlock use of multithreading was essential for this project.

Team Project - database driven application "GARITS"

With a group of 4 other students I had to create an application for the imaginary carshop planing to switch to automated management system - GARage IT System - GARITS.

This program was supposed to have different functions for different types of users (mechanic, receptionist, foreperson, franchisee and admin). However, in essence, it was supposed to:

  • collect and save customers' details
  • create invoices in PDF format
  • give the ability to manage stocked parts (order and sell parts)
  • schedule car services
  • generate reports on aspects like stock level or carried out jobs
  • calculate prices and prepare discounts for customers
Apart from the application itself, we were supposed to prepare many diagrams describing our system. Among others, we had to create Class Diagram, Use-Case Diagram and Entity-Relation Diagram. All of them were done in the Visual Paradigm program.

As I was the person that felt most comfortable with programming, apart from standard coding, I took care of the more unusual or harder tasks like developing part of the program that dynamically creates PDF documents. I have also established a connection with database and developed functionality that allows to back up and resore it. Additionally, I worked on the harder GUI forms and programmed the ability for the user to change themes of the GUI from light to dark.

Programming and Mathematics for Artificial Inteligence

This project was a coursework summarising my work throughthout the module "Programming and Mathematics for Artificial Inteligence". It consisted of three parts, which were to demonstrate my abilities in terms of Programming, Neural Networks, and Deep Learning.

Shortest path algorithms

The goal of this task was to develop a simple game consisting of a rectangular grid (of size height x width) where each cell had a random value between 0 and 9. An agent would start at the upper-left corner of the grid and had to reach the lower-right corner of the grid as fast as possible. I had to implement a couple of algorithms to find the shortest path and compare their performance against each other.

Development of Neural Network

This part of coursework was about classification on the MNIST dataset. I had to develop fully parameterizable neural network class without the use of libraries like PyTorch or TensorFlow. I was supposed to:

  • Implement sigmoid and relu layers
  • Develop a softmax output layer
  • Implement Dropout or regularization for all layers
  • Create a test suite for different layers
Convolutional Neural Network

For this task I had to implementing two types of neural networks. First was the convolutional NN using PyTorch for a dataset (Fashion-MNIST). It was supposed to analise provided images and determin which piece of clothing they represented. The second NN was also a classification task, however it had to determin the possibility of rain in Australia based on meteorological data.
